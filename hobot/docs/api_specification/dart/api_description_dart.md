# Open DART API: 다중회사 주요계정

이 API는 여러 회사의 주요 계정(재무상태표, 손익계산서의 핵심 항목) 정보를 한 번에 조회할 수 있는 기능을 제공합니다.

## 1. 기본 정보

| 항목 | 내용 |
| --- | --- |
| **API 명칭** | 다중회사 주요계정 |
| **서비스 ID** | DS003 (2019017) |
| **요청 URL** | `https://opendart.fss.or.kr/api/fnlttMultiAcnt.json` (또는 `.xml`) |
| **메서드** | GET |
| **출력 포맷** | JSON, XML |
| **인코딩** | UTF-8 |

## 2. 요청 인자 (Request Parameters)

| 키(Key) | 명칭 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- | --- |
| **crtfc_key** | API 인증키 | STRING(40) | Y | 발급받은 인증키 (40자리) |
| **corp_code** | 고유번호 | STRING | Y | 공시대상회사의 고유번호 (8자리)<br>

<br>※ 콤마(`,`)로 구분하여 다수 입력 가능 |
| **bsns_year** | 사업연도 | STRING(4) | Y | 사업연도 (예: 2023) <br>

<br>※ 2015년 이후 정보부터 제공 |
| **reprt_code** | 보고서 코드 | STRING(5) | Y | `11013`: 1분기보고서<br>

<br>`11012`: 반기보고서<br>

<br>`11014`: 3분기보고서<br>

<br>`11011`: 사업보고서 |

## 3. 응답 결과 (Response)

### 3-1. 공통부

| 키(Key) | 명칭 | 설명 |
| --- | --- | --- |
| **status** | 에러 및 정보 코드 | `000`: 정상 (하단 에러 코드 참조) |
| **message** | 에러 및 정보 메시지 | 정상 또는 에러 내용 출력 |

### 3-2. 데이터 목록 (`list`)

| 키(Key) | 명칭 | 설명 |
| --- | --- | --- |
| **rcept_no** | 접수번호 | 접수번호 (14자리) |
| **bsns_year** | 사업연도 | 사업연도 (4자리) |
| **stock_code** | 종목 코드 | 상장회사의 종목코드 (6자리) |
| **reprt_code** | 보고서 코드 | 요청 시 입력한 보고서 코드 (예: 11011) |
| **account_nm** | 계정명 | 예) 자본총계, 매출액 등 |
| **fs_div** | 개별/연결 구분 | `OFS`: 재무제표 (개별)<br>

<br>`CFS`: 연결재무제표 |
| **fs_nm** | 개별/연결명 | 예) 연결재무제표 또는 재무제표 |
| **sj_div** | 재무제표 구분 | `BS`: 재무상태표<br>

<br>`IS`: 손익계산서 |
| **sj_nm** | 재무제표명 | 예) 재무상태표 또는 손익계산서 |
| **thstrm_nm** | 당기명 | 예) 제 13 기 3분기말 |
| **thstrm_dt** | 당기일자 | 예) 2018.09.30 현재 |
| **thstrm_amount** | 당기금액 | 당기 금액 (단위: 원) |
| **thstrm_add_amount** | 당기누적금액 | 당기 누적 금액 |
| **frmtrm_nm** | 전기명 | 예) 제 12 기말 |
| **frmtrm_dt** | 전기일자 | 예) 2017.01.01 ~ 2017.12.31 |
| **frmtrm_amount** | 전기금액 | 전기 금액 |
| **frmtrm_add_amount** | 전기누적금액 | 전기 누적 금액 |
| **bfefrmtrm_nm** | 전전기명 | 예) 제 11 기말 (※ 사업보고서의 경우만 출력) |
| **bfefrmtrm_dt** | 전전기일자 | 예) 2016.12.31 현재 (※ 사업보고서의 경우만 출력) |
| **bfefrmtrm_amount** | 전전기금액 | 전전기 금액 (※ 사업보고서의 경우만 출력) |
| **ord** | 정렬순서 | 계정과목 정렬 순서 |
| **currency** | 통화 단위 | 통화 단위 (예: KRW) |

## 4. 에러 및 메시지 코드 설명

| 코드 | 설명 |
| --- | --- |
| **000** | 정상 |
| **010** | 등록되지 않은 키입니다. |
| **011** | 사용할 수 없는 키입니다. (일시 사용 중지 등) |
| **013** | 조회된 데이터가 없습니다. |
| **020** | 요청 제한을 초과하였습니다. (일반적으로 20,000건/일) |
| **021** | 조회 가능한 회사 개수가 초과하였습니다. (최대 100건) |
| **100** | 필드의 부적절한 값입니다. |
| **800** | 시스템 점검으로 인한 서비스 중지 중입니다. |

> **참고:** 이 API는 전체 재무제표가 아닌 주요 계정(유동자산, 부채총계, 자본총계, 매출액, 영업이익, 당기순이익 등) 위주로 데이터를 제공합니다.
