# Phase5 회귀 알림 연동 계획

## 목표
- Phase 5 GraphRAG 일일 회귀 실행 결과를 장 마감 이후 운영 관점에서 빠르게 인지할 수 있도록 알림 채널(Slack) 연동.
- 기존 스케줄/리포트 저장 로직은 유지하고, 알림은 옵션(env) 기반으로 안전하게 동작.

## 범위
1. `scheduler.py`에 Phase5 알림 전송 헬퍼 추가
2. `run_graph_rag_phase5_regression` 성공/실패 경로에 알림 호출 연결
3. 단위 테스트 보강
4. 워크플로우 기록 업데이트

## 설계 원칙
- 기본값은 알림 비활성(`GRAPH_RAG_PHASE5_ALERT_ENABLED=0`)
- Slack 토큰/모듈 문제로 본 작업 흐름이 중단되지 않도록 예외를 모두 흡수하고 경고 로그만 기록
- 실패 케이스 요약을 짧게 포함해 원인 파악 시간을 줄임
